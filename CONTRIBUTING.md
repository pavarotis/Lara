Contributing to Lara

Σας ευχαριστούμε που θέλετε να συμβάλετε στο Lara!
Η συμμετοχή σας βοηθάει το project να εξελίσσεται και να παραμένει ποιοτικό.

Ακολουθήστε τις παρακάτω οδηγίες για να διατηρούμε ένα σταθερό και οργανωμένο workflow.

🧩 Πώς να συμβάλετε
1. Δημιουργήστε ένα issue

Πριν ξεκινήσετε οποιαδήποτε αλλαγή:

Ανοίξτε ένα Issue

Περιγράψτε ξεκάθαρα το bug, feature ή βελτίωση

Προτείνετε προσέγγιση (optional)

Με αυτόν τον τρόπο συντονίζουμε την ανάπτυξη.

2. Κάντε Fork & Branch

Κάντε fork το repository

Δημιουργήστε νέο branch με περιγραφικό όνομα:

feature/my-new-feature
fix/issue-123
refactor/module-users


Μην κάνετε αλλαγές απευθείας στο main.

3. Coding Standards

Ακολουθήστε τα παρακάτω standards:

PHP: PSR-12

Laravel best practices

Consistent naming

Self-documented κώδικας

Χωρίς debug code (dd(), dump(), var_dump())

Τηρείτε τη δομή αρχείων του project

4. Tests

Όπου είναι δυνατόν:

Προσθέστε ή ενημερώστε unit/feature tests

Βεβαιωθείτε ότι όλα τα tests περνάνε:

php artisan test

5. Commit Messages

Γράψτε καθαρά, κατανοητά commits:

feat: add user roles management
fix: correct auth middleware bug
refactor: clean up user controller
docs: update readme

6. Pull Request (PR)

Όταν είστε έτοιμοι:

Ανοίξτε PR προς το main

Περιγράψτε:

το τι αλλάξατε

το γιατί

σχετικά issues

Βεβαιωθείτε ότι:

περνάνε όλα τα tests

ο κώδικας είναι καθαρός

η αλλαγή είναι μικρή & εστιασμένη

Οι μεγάλοι PRs αποφεύγονται.

🧪 Review Process

Κάθε PR ελέγχεται από maintainers

Μπορεί να ζητηθούν αλλαγές

Το merge γίνεται μόνο όταν πληρούνται τα standards

❤️ Ευχαριστούμε!

Η συμβολή σας βοηθάει το Lara να γίνεται καλύτερο και πιο αξιόπιστο.
Καλή ανάπτυξη!
